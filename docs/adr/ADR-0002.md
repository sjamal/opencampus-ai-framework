# ADR-0002: Python Tooling

- **Status:** Accepted
- **Date:** 2026-07-07

## Context

The project requires a modern, reproducible Python development environment that is easy to set up, isolated from the host operating system, and suitable for both local development and future CI/CD pipelines.

## Decision

The project will adopt:

- Python 3.14+
- `uv` for project initialization, virtual environment management, and dependency management
- A repository-local virtual environment (`.venv`)
- `pyproject.toml` as the single source of project configuration
- Visual Studio Code as the reference development environment

## Rationale

### Python

Python provides a mature ecosystem for:

- Enterprise APIs
- AI/LLM integration
- MCP development
- Data processing
- Automation

### uv

`uv` offers significantly faster dependency resolution and project management than traditional `pip` workflows while remaining compatible with standard Python packaging.

### Repository-local virtual environment

A local `.venv`:

- isolates dependencies
- simplifies VS Code integration
- avoids conflicts between projects
- improves reproducibility

### pyproject.toml

Using a single project definition aligns with current Python packaging standards and reduces configuration duplication.

## Consequences

### Advantages

- Modern tooling
- Faster dependency management
- Consistent developer experience
- Simplified onboarding

### Trade-offs

Developers unfamiliar with `uv` may require a short learning period, although the workflow remains compatible with standard Python practices.
