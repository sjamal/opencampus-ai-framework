# Security Policy

Security is a foundational design principle of the OpenCampus AI Framework.

## Objectives

The framework is being designed with:

- Secure authentication
- Authorization
- Audit logging
- Least privilege
- Secure secrets management
- Responsible AI integration


## Supported Versions

The project is under active development. Security fixes are applied to the latest version on the `main` branch.

## Reporting Security Issues

Please do not disclose security vulnerabilities publicly.

As the project matures, a responsible disclosure process will be documented here.

## Reporting a Security Issue or Vulnerability

Please do not disclose security vulnerabilities publicly or file a public GitHub issue.

Instead, contact the project maintainer privately with:

- A description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

Reports will be acknowledged promptly and investigated before public disclosure.

## Current Status

The project is in active development and should not be considered production-ready.
